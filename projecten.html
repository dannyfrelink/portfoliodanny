Namaak Lay's website

Voor deze opdracht ben ik de gehele Lay's website gaan namaken in code. Om de site op te breken ben ik begonnen met het maken van een Breakdown schets van de site.

Mobiele versie

Nadat ik een beeld had van hoe de website in elkaar zat, ben ik begonnen door Mobile First te werken. Al gauw had ik mijn mobiele versie vrijwel identiek aan het origineel.

Responsive

Vanuit mijn mobiele versie ben ik verder gegaan door stapsgewijs code toe te voegen, wanneer ik de grootte van het scherm aanpaste. Hiermee kwam ik op mijn eindresultaat voor mijn productenpagina van de Lay's website.

Detailpagina

Bij deze opdracht hoorde ook nog een uitgewerkte detailpagina. In mijn eerste design is het origineel van de Lay's website te zien. Bij de twee andere foto's is het gebruik van backface-visibility terug te zien.

Desing TripPlaner website

Mijn eigen reiswebsite design, waarbij de opdracht was om een multi-device ontwerp te maken voor 2 fases tijdens de reis. Ik heb hierbij gekozen om voor een laptop en mobiel te kiezen in de fases voor en tijdens de reis.

Onderzoeksfase

Ik begon door te kijken naar in welke fase welk device wordt gebruikt en waarvoor deze wordt ingezet. Vervolgens ben ik verder gegaan met het maken van wireflows voor mijn website.

Eerste versie

Voor mijn eerste versie ben ik begonnen met het maken van de schermen die ik in mijn wireflows heb gemaakt. Hierbij heb ik de fases die horen bij mobiel en desktop formaat uitgewerkt, zodat ik geen onnodig dubbel werk lever.

Tweede versie

Vervolgens ben ik verder gaan uitbreiden door meer pagina's toe te voegen aan mijn ontwerp. Ook ben ik hier al begonnen met het toevoegen van details, zoals een loading state, hamburger menu, etc.

Eindresultaat

Hiervoor heb ik puur de puntjes op de i gezet en heb ik voornamelijk gewerkt om mijn ontwerp gebruikervriendelijk te maken. Zo heb ik een disabled/enabled state, locatievoorzieningen, etc. en een informatie pagina toegevoegd.
